# Contributing to awesome-ai-security

This repository is meant to help, not harm -- please act accordingly by only putting in PRs for tools and resources that you have knowledge of. 

Maintainers recognize that there is a middle ground. In this emerging area there are many voices and many projects, some more useful than others.

We are already watching for contributors astroturfing the repository with random resources or low-quality resources.

There are some resources that are popular that are not necessarily useful right now or that may contain hyped messages or errors. 

Use your best judgement and ask yourself whether they are worth including.

## PRs and Issues

If you know how to create PRs, use PRs. If not, Issues are fine -- describe what you would like to see. If you can't use either reach out to a maintainer. 

1. Put in a PR/issue for the resource change that you think is awesome
2. Tell us in the PR/issue why you know it is awesome.

## Resource vetting
For new resources we look at these and other criteria to find out if it's truly awesome:

* Does the resource help the intended audience -- security, operations, and risk professionals?
* Does the resource have users today that find it useful? 
* Does the resource, even if flawed, have a demonstrated industry-wide impact?
* Is the resource from a leading expert with applied practical skill demonstrated by their track record?
* Is the resource seminal -- it set a new standard or inspired a new conversation in the AI security space?
* Is the resource up to date? (some may not be updatable, e.g. reports, but that's a case-by-case)
